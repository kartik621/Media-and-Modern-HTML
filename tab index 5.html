Short Note on Tab Index

Tab index is an attribute used in web development to specify the order in which elements can be accessed using the keyboard's
Tab key. It determines the focus navigation sequence when users navigate through a web page using the keyboard, allowing
them to move between interactive elements easily.

The tab index attribute can be applied to various HTML elements such as links, form controls (input fields, checkboxes, radio
buttons, etc.), buttons, and other focusable elements. It assigns a numerical value to each element, indicating its position in the
tabbing order. The value of the tab index attribute can be positive, zero, or negative.

- A positive tab index (e.g., 1, 2, 3) indicates the element will be included in the default tab order and determines the sequence
in which it receives focus when the Tab key is pressed. The element with the lowest positive tab index value is the first to receive
focus, followed by the element with the next higher value, and so on.

- A zero tab index (tabindex="0") allows an element to be included in the default tab order based on its position in the HTML
document's source order. It behaves like a positive tab index but is typically used to make non-interactive elements, such as
divs or spans, focusable for accessibility purposes. This can be useful when adding interactive behavior to elements through
JavaScript.

- A negative tab index (e.g., -1) excludes an element from the default tab order, meaning it cannot be reached by pressing the
Tab key. However, it can still receive focus programmatically using JavaScript. This is often used for elements that are hidden,
but should still be accessible to assistive technologies or for elements that have alternative ways of receiving focus (e.g.,
through JavaScript events).

It's important to note that relying solely on tab index for navigation can lead to accessibility issues if not implemented correctly.
It's recommended to follow proper HTML structure and semantics, ensuring elements are ordered logically in the source code.
This allows users to navigate the page naturally and intuitively using the Tab key, without needing to rely on modifying the tab
index manually.
